## Prioritized findings (severity-ranked)

### HIGH
1. **No pagination/virtualization on ~all list screens** (only Procurement has it). At ZZ volume — Commodity **1000**, MTO Detail **~3,338 lines**, Equipment 400, Stock ~800, Traceability ~300 — these render every row. **Lag/freeze risk** on load/filter/sort. *Manual: time initial load + a sort on Commodity & MTO Detail at ZZ volume.*
2. **WBS Gantt at "All" depth + ~500 nodes** — render/scroll/perf + controls reachability unverified; `vw/vh` in `WBSGanttView` → zoom-clip risk. (Plus the still-open **A1 Tree depth-filter leak**, already logged.) *Manual.*
3. **Modal clipping at Large zoom (the dad8b99 class)** — many modals use `vw/vh` (`CertificatesModal`, `CreateSCNWizard`, `CertDetailModal`, `ReviewCertModal`, `ChangePasswordModal`, etc.). High-risk: any maximized modal + MC Receipting stock-take. *Manual at Large (1.15).*

### MED
4. **A1 — variation read-side not aligned.** LogisticsScreen shows variations as generic "Additional item" with no parent/`is_variation` label; backend GET doesn't join the parent ref. Wizard creation ≠ display. *(Fix: join parent line in the GET + render "Off-PO variation — for: Line N".)*
5. **Resizable-tables rollout gap** — present only on Procurement, Admin, ConfirmerQueue (+ Dashboard via the older `useTableResize`); **missing on ~15 screens** (WBS, Commodity, Equipment, MTO×2, PO Detail, Expediting×2, Logistics, MC×4, Traceability, Document Inbox). Matches the logged standing-rule rollout track.
6. **Help (`?`) missing** on Logistics, MC Receipting/Stock/FMR/Transfers, Traceability, Document Inbox, ConfirmerQueue (matches the unfinished Ch8–11 manual/help pass).
7. **Legends missing where colour carries meaning** — MC stock condition pills, Traceability hold colours, MC status colours, PO Detail, ConfirmerQueue. First-time users can't decode the colours.

### LOW / INFO
8. A3 child-items vs variations — different screens; confirm visually distinct (low risk).
9. A4 VDRL — no `vdrl` seed on project 9; needs seeded data to audit.
10. Dashboard transient "Network Error" — recheck on reload (backend was mid-restart; `/api/projects` returns 200).
